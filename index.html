About the Project

Healthcare Accessibility

The GeoHealth Web Application aims to analyze the accessibility of healthcare facilities in Kathmandu Valley by using geospatial data. It leverages the power of the GeoDjango framework to integrate hospital location data, calculate Euclidean distances in projected space, and visualize healthcare accessibility on interactive maps.

Interactive Features

The application allows users to input a specific radius in meters and find the nearest hospitals within that radius. Additionally, buffer zones are displayed to show the coverage area, along with key base and overlay maps that enhance the user experience.

Framework Used

The application uses the GeoDjango framework, which extends Django to support geospatial data. GeoDjango makes it possible to work with geographic information systems (GIS) data and create powerful spatial queries and visualizations.

Base Layers

The application integrates the following base layers to provide detailed maps:

  • OpenStreetMap
  • Esri Imagery
  • Carto Light
  • Dark Matter

Overlay Layers

The following overlay layers are used to enhance the map's interactivity:

  • Kathmandu Valley Shapefile
  • Kathmandu Hospital Data
  • Weather API Group Layer

Frontend Design

The application utilizes an intuitive frontend design using JavaScript and CSS, ensuring smooth interactions. The hospital-themed background image reflects the core focus of the application, providing a clear visual association with healthcare accessibility.

CQL Filters

The application uses CQL (Common Query Language) filters to refine search results. Users can filter hospitals by type, services offered, or location. This makes it easier to find the best healthcare options based on specific needs.

  • Filter by hospital type: "TYPE = 'General Hospital'"
  • Filter by services: "SERVICES LIKE '%Emergency%'"
  • Filter by location: "INTERSECTS(geom, POLYGON((...)))"

Bootstrap Integration

The application uses Bootstrap to build responsive and mobile-first web pages. It helps in creating a flexible and user-friendly interface, making the application accessible across different devices and screen sizes.
